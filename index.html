航港大數據比賽

人流與貨流交互影響分析之決策支援系統提案

先看數據庫 → 找可以分析的數據 (考量數量跟可否做正規化) → 用數據推估問題 → Data 處理 (資料變資訊) → 資訊理解 → 提出定義明確問題 → Solution

動機

在全球港口運作中，大型商港多呈現「多功能混合型態」，同時承載貨運、客運與其他港區活動。以我們台灣的高雄港來說，其具備高度貨櫃吞吐能力，同時亦有郵輪停靠與旅客流動，這樣的大型混用港口會形成人流與貨流並行的運作環境。

過去航港相關研究多聚焦於單一面向，例如貨櫃調度最佳化、船舶排程或碳排放分析，較少從「人流與貨流交互影響」的角度切入。然而，或許在特定時間區間（如郵輪靠港時段或多船舶集中進港時），人流與貨流可能出現時間上的重疊，進而對港區交通動線、作業節奏與資源配置產生壓力。

在此背景下，隨著航港資料與政府開放資料逐漸完善，本研究嘗試從大數據分析角度出發，透過多源資料整合，觀察港口在不同時段之人流與貨流分佈特性，進一步辨識潛在的高負載情境。

長話短說：

用航港資料跟政府開源資料去挖「原先假定關於人流跟貨流在時間空間的交錯分布關係」，再分析交錯重疊的情境對港口運作可能會造成的潛在影響，進而提出解法。

資料來源與資料運用 (DB)

用比賽網頁提到的 iMarine、政府開放資料。之後要做跨資料庫整合跟分析。

1. iMarine 船舶運作資料

  • Key: 船舶進出港時間 (Arrival/Departure Time)、船舶類型 (貨船、郵輪等)、船舶名稱/IMO (識別用)、停靠碼頭/港區位置、航次資訊。
  • Use: 建立「貨流」時間分佈、區分不同船型 (貨船 vs 郵輪)、分析船舶集中時段。

2. data.gov.tw 觀光資料

  • Key: 郵輪旅客人次、港口旅運統計。
  • Use: 觀光人流補充（資料分析切入點）、建立「人流」估計模型（將旅客數量轉換為時間分佈，例如靠港後展開）。

3. 宏觀資料方向 (DB 待定)

  • Key: 港口吞吐量、貨物流量統計、運輸相關指標、貨物特性資料、貨物損耗或延誤統計。
  • Use: 補強貨物流規模背景、作為 Normalization 參考、建立「貨物優先性 (Priority)」、支援決策邏輯。

預期分析方式 (Approach)

時間分佈建構

  • 將船舶進出港資料轉為「每小時貨流量」。
  • 將旅客資料轉為「每小時人流量 (估計)」。

人流 × 貨流重疊分析

辨識：人流高峰、貨流高峰、同時出現之時間區間。

定義高負載情境：高重疊時段、潛在壅塞風險區間。

核心研究問題

多功能混合港口運作過程中，人流與貨流於特定時間區間之重疊，是否形成潛在運作壓力，並可否透過資料分析加以辨識？

研究方向與呈現：

  1. 理解資料分布特性。
  2. 建立風險認定的指標（人貨重疊與風險的關係）。
  3. 針對「聚焦完成的問題」延伸應用面。

決策支援：不做全域最佳解（對參數調用跟數學要求極高），而是做「條件下最佳解」。從數據找出問題，提供解法跟成效，成效可以用歷史數據回測。公式可拓展成 Dashboard 的 Input，避開部署即時 Server 的壓力與複雜預測。

資料處理與指標建構

Data Preprocessing

  • 統一的 Timestamp 與 Data Table Alignment。
  • Timestamp Normalization。
  • 去除異常、缺樣 Data。
  • 船種類 Label (人、貨、軍、郵等等)。
  • 時間展開 (Temporal Disaggregation)：把用「總量」記錄的 Data 轉換成「Time Series」的做法。常用方法包括平均法或分段分配 (如 6:4 比)。

Temporal Distribution Modeling

  • 貨流：以船舶進出港資料為基礎（每單位時間如每小時的貨船數量）。
  • 人流：以郵輪旅客數為基礎，進行 Time Spread，建立人流時間分佈曲線。
  • 目標：建立港口在不同時間區間的人流與貨流強度分佈。

建構人貨流重疊與風險指標

Overlap 指標：衡量人流與貨流同時出現之程度。

  • Method: 用人流與貨流資料做 Normalization，選擇合適交互強度公式（如乘積或加權）。
  • Intention: 判斷人貨流是否同時處於高峰、重疊強度。

壅塞指標 (Risk 層級)：

  • 公式：貨流 x 人流 x Overlap 程度。
  • Intention: Data to Information，辨別高重疊時段、潛在壅塞窗口、特定日期或事件。分析其頻率 (Frequency) 與規律性 (Regularity)。

Decision Support Layer (決策支援層)

針對資料分析結果建構風險標準，將 Input 變成決策條件，並針對多情境 (多變數) 下做建議。

納入決策因素：

  • 貨物優先性 (緊急/一般/有期效)。
  • 人流影響程度 (旅客量)。
  • 港口負載狀態 (風險等級)。
策略類型 說明
效率優先 優先貨運作業
旅客優先 優先人流疏運
風險降低 避免高重疊情境

Visualization & Dashboard

理想是「半互動決策儀表板」，讓使用者輸入部分資訊進行觀察。

  • Input Info: 日期時間選擇、策略切換 (人優先/貨優先/50-50)、貨物等級。
  • 情境模擬: 例如增加貨流對風險的影響。
  • 呈現方式: 人貨流隨時間單位變化量、Overlap Heatmap、風險層級、決策建議。

題目優勢與價值

  • 優勢：Data-driven (數據驅動)、Explainable (具解釋性)、Scalable (具擴充性)。
  • 價值：理解人貨流動態關係、快速辨識高風險區間、比較不同情境下的決策結果、以視覺化支援港口管理。
