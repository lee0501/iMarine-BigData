零、背景動機

（一）相關報導與現象

  1. 《藍色公路獎勵政策》推動空櫃海運調度
    自民國102年起，政府持續推動「藍色公路獎勵」，鼓勵業者以海運取代陸運進行空櫃調度，作為減碳與運輸優化措施。至113年（2024年），透過該政策促成之貨櫃裝卸量已達130萬 TEU，顯示政府已積極介入空櫃調度問題，並試圖透過誘因機制改變運輸行為。
  2. 航運公司傾向自行持有或租用貨櫃，降低共享意願
    以長榮海運為例，其以約新台幣15億元租用25,000只貨櫃，租期長達7年，反映出航商在面對空櫃需求不確定性時，傾向透過自行掌控資源來降低風險，而非依賴外部調度或共享機制。
  3. 既有空櫃媒合多為港口互換
    過去曾出現空櫃媒合平台-NCS，其運作方式主要為在同一港口內進行雙邊交換（某公司缺櫃、另一公司多櫃），以達到即時調度。
  4. 國際實務（One Way Use）仍存在責任與風險問題
    國際上亦有「One Way Use」模式，即單次租用空櫃。
  5. 第七貨櫃中心全面落成/營運，智慧高港翻轉無限可能
    港口轉型推高整體港口競爭力及全球貨櫃港吞吐量，維持高雄港亞太地區貨櫃轉運樞紐地位

（二）產業背景說明

在全球與區域航運體系中，空櫃失衡（empty container imbalance）為長期存在之營運現象，其主要特徵並非空櫃總量不足，而是空櫃在不同港口與區域之間的分布不均。此種不均衡會使部分港口出現空櫃累積、堆場壓力上升，而其他港口則可能處於相對低壓或空櫃供給不足的狀態。

以台灣港口體系為例，南北港口之間在不同時期可能出現空櫃配置差異，實務上常透過陸運進行調度，形成「南櫃北運」或「北櫃南運」的情形。此類調度方式雖可短期平衡供需，但同時也伴隨運輸成本增加、交通負荷與環境外部性等問題。

此外，空櫃管理在實務上涉及多方角色（航運公司、港務單位、物流業者），且貨櫃調度決策往往與船期安排、艙位利用及營運策略相互影響，使得空櫃流動不易由單一主體進行整體優化配置。在缺乏跨港整體視角與量化依據的情況下，空櫃調度多依賴經驗判斷或臨時協調進行。

過去模式（港口互換、單次租賃等等） 多僅解決「當下最優」，缺乏跨港、跨時間的整體視角與配置能力，也未能有效解決航商間對於責任歸屬與風險分擔的疑慮。

一、研究動機

全球與區域航運體系中，空櫃失衡（empty container imbalance）為長期存在之營運問題。其本質並非空櫃總量絕對不足，而是空櫃在不同港口之間配置不均，導致部分港口空櫃持續累積、堆場壓力升高，另一些港口則維持較低壓力狀態。

可分為兩個狀態:

空櫃不足： 出口需求旺盛但空櫃供給不足，導致貨物無法即時裝櫃出運，影響供應鏈效率與港口競爭力（尤其台灣是「出口導向」國家，可以引GDP資料佐證）

空櫃過多： 空櫃滯留港區，占用堆場與作業空間，降低港口運作效率並增加管理負擔

在台灣港口情境下，當某些港口空櫃量過高時，若未能即時外移，將可能造成堆場空間占用、作業效率下降，甚至形成空櫃滯留（死櫃）問題。現行實務多仰賴航商與港務單位以經驗判斷進行協調，較缺乏一套以公開資料為基礎的量化辨識與決策支援方法。

因此，本研究擬建立一套以港口壓力釋放為核心之空櫃失衡辨識與跨港媒合決策機制，協助政府與港務公司辨識需釋放空櫃之港口，以及具有承接能力之港口，進而提升空櫃跨港調度效率與決策品質。

補充-空櫃失衡（empty container imbalance）
空櫃在「不同港口之間配置不均」，概念上是**多個港口**都在面對空櫃所導致的過多或過少的狀態，並且缺乏有效的調度*。
補充-空櫃管理責任分散
「重櫃運送及空櫃調度則常是分屬不同部門管理，實務上除非櫃務部門提出空櫃運送需求，否則空櫃皆是等到有剩餘艙位才思考是否運送，時常有累積過多貨櫃的情況發生，此情形對於船公司造成浪費」 （來源：李際偉、許家瑋：定期航運業艙位分配及空櫃調度之研究 海運學報 第二十卷 第二期 「摘要」）

二、研究核心問題

不是主打「預測」（這樣可以避開要用時間模型的做法，去年社會組第一名就是用時間模型的做法），而是主打「資源配置 / 調度問題」


本研究欲回答以下三個核心問題：

  1. 如何利用公開航港資料辨識哪些港口已出現空櫃壓力過高、需優先釋放空櫃？
  2. 如何辨識哪些港口仍具有空間緩衝與承接能力，可作為空櫃外移之候選承接端？
  3. 如何建立跨港調度媒合與優先排序機制，提供政府與港務公司作為決策支援依據？

三、研究邏輯與整體架構

3.1 研究邏輯

本研究不以航商訂單或需求預測作為分析主軸，而採用 push-based 調度邏輯。也就是說，研究起點不是「哪個港口需要空櫃」，而是「哪個港口的空櫃已累積到必須外移」。在此邏輯下，研究將依序完成以下任務：

  1. 辨識需釋放空櫃之港口：找出空櫃壓力過高、已接近或超過安全區間的港口。
  2. 辨識具承接能力之港口：找出當前空櫃壓力較低、仍有承接緩衝空間的港口。
  3. 建立跨港候選組合：將高壓力港與高緩衝港進行配對，形成潛在調度對。
  4. 進行媒合排序：依據壓力程度、承接能力與調度阻力等因素，形成調度優先順序。
  5. 進行視覺化呈現：將結果整合為儀表板，作為管理與溝通工具。

3.2 系統架構分層

  • Layer 1：資料層
    整合 iMarine、政府資料開放平台及主計總處之港口與貨櫃月別資料。
  • Layer 2：指標建構層
    將原始資料轉換為可判讀之港口壓力、空櫃流動與承接緩衝指標。
  • Layer 3：分析與分類層
    判斷各港口目前屬於高壓力港、正常港或高緩衝港。
  • Layer 4：決策支援層
    建立跨港媒合候選對與排序規則，輸出調度優先建議。
  • Layer 5：視覺化呈現層
    以圖表與互動式介面呈現港口狀態、失衡趨勢與調度建議。

四、資料來源與資料用途

本研究以「港口 × 年月」為分析主軸，整合空櫃流動、實櫃活動與港口作業量資料，建立可用於辨識壓力、緩衝與媒合決策的月別主表。本章說明資料在研究中的角色分工；各資料集的實際連結、欄位與轉化方式集中整理於第十章。

4.1 核心資料：空櫃流動

  • 進口空櫃資料：用來描述空櫃流入港口的規模，是觀察空櫃是否持續堆積的起點。
  • 出口空櫃資料：用來描述空櫃釋放或外移的規模，搭配流入量可計算空櫃淨流量。
  • 分析角色：支撐 empty_in、empty_out 與 empty_net 等核心指標建構。

4.2 輔助資料：實櫃活動

  • 進口實櫃資料：補充港口整體進口貨物流動狀況。
  • 出口實櫃資料：反映港口出口活動強度與貨物流規模。
  • 分析角色：作為背景解釋與合理性驗證，不直接作為主決策排序依據。

4.3 壓力基準資料：港口作業量

  • 貨櫃裝卸量（TEU）：用來估計各港口的作業規模與負荷程度。
  • 分析角色：作為壓力正規化基準，避免不同規模港口直接以絕對櫃量比較。

4.4 備援與驗證資料

  • 官方統計資料：用於交叉驗證資料一致性，並在原始來源缺漏或異常時提供備援。
  • 分析角色：提升月別資料整合後的穩定性與可信度。

4.5 最終整合方向

整體資料將彙整為單一分析表，欄位涵蓋港口、年月、空櫃進出、實櫃進出與港口作業量，供後續進行壓力辨識、港口分類、跨港媒合排序與視覺化呈現。

五、研究方法

5.1 資料處理與清洗

  1. 欄位命名統一：將不同資料表的港口名稱、年月、櫃量欄位標準化。
  2. 港口名稱標準化：統一港口命名方式，以免同港異名造成 join 錯誤。
  3. 年月格式統一：將年月欄位轉為一致格式，例如 YYYY-MM。
  4. 單位統一：統一以 TEU 為分析單位，避免不同統計口徑造成比較偏差。
  5. 缺失值檢查與處理：檢查空值、異常值或重複值，視情況採刪除、補值或註記。
  6. 建立月別主表：以 港口 × 年月 為主鍵，將各資料表合併。

5.2 指標建構

（1）空櫃流動構面

  • 空櫃進港量
  • 空櫃出港量
  • 空櫃淨流量
  • 空櫃累積趨勢

（2）港口壓力構面

反映空櫃在港口整體作業中造成的壓力程度。重點是「相對於港口作業規模或歷史狀態，是否已偏高」。

（3）承接緩衝構面

衡量港口尚可承接外移空櫃的空間。代表該港在現況下不致因承接而快速進入高壓力狀態。

5.3 港口狀態分類

將港口區分為：高壓力港、正常港、高緩衝港。分類方式採相對門檻設計（如歷史分位數、z-score 或相對容量門檻）。

5.4 跨港媒合與決策排序

採 push-based 調度邏輯，流程如下：

  1. 辨識高壓力港（釋放端）。
  2. 辨識高緩衝港（承接端）。
  3. 建立跨港候選組合。
  4. 進行排序（考慮壓力程度、緩衝程度、距離阻力等）。

目標是建立具實務可解釋性之 調度優先順序清單。

5.5 視覺化與系統呈現

使用 R 語言之 ggplot2 與 Shiny 預期包含：壓力現況圖、失衡趨勢圖、承接能力比較圖、候選調度對應表、優先順序建議表。

七、預期呈現模式（視覺化與系統呈現）

本研究將分析結果整合為一套以決策支援為導向之互動式儀表板，協助使用者快速掌握港口空櫃狀態與跨港調度建議。視覺化內容將分為四個層次：

7.1 港口空櫃壓力現況（Status Overview）

目的在於快速辨識目前哪些港口處於高壓力狀態。

視覺化方式：

  • 長條圖（Bar Chart）：比較各港口當月空櫃壓力指標
  • 顏色標示（Color Encoding）： 紅色：高壓力港（需釋放）； 灰色：正常港； 綠色：高緩衝港（可承接）

呈現內容：

  • 各港口空櫃壓力排名
  • 高壓力港清單
  • 高緩衝港清單

7.2 空櫃失衡趨勢分析（Trend Analysis）

目的在於觀察各港口空櫃是否存在「持續累積」或「持續流失」現象。

視覺化方式：

  • 折線圖（Line Chart）：呈現各港口空櫃淨流量（Empty In - Empty Out）隨時間變化
  • 累積曲線（Cumulative Curve）：觀察長期堆積或流失趨勢

呈現內容：

  • 哪些港口長期為正（累積風險）
  • 哪些港口長期為負（流失狀態）
  • 是否存在季節性波動

7.3 承接能力分析（Absorption Capacity）

目的在於辨識哪些港口具有承接外移空櫃之潛力。

視覺化方式：

  • 散佈圖（Scatter Plot）：X 軸為港口壓力指標，Y 軸為承接緩衝指標（Buffer Index）

呈現內容：

  • 右上角：高壓力港（需釋放）
  • 左上角：低壓力＋高緩衝港（最佳承接端）
  • 視覺上直接區分「可承接區域」

7.4 跨港調度媒合建議（Matching & Recommendation）

目的在於提供具體可執行之調度建議。

視覺化方式：

  • 網絡圖（Network Graph）：節點為港口，邊為建議調度方向（例如 高雄 → 台中）
  • 表格（Ranking Table）：呈現調度來源港、承接港、媒合分數（Score）與優先順序

呈現內容：

  • Top-N 調度建議
  • 每個高壓力港對應之候選承接港
  • 調度優先順序

7.5 情境模擬（Simulation）（選配加分）

目的在於讓使用者理解「若不調度」與「調度後」的差異。

視覺化方式：

  • Before / After 比較圖
  • 滑動控制（Slider）模擬不同調度量

呈現內容：

  • 若不調度 → 壓力持續上升
  • 若調度 → 壓力下降幅度
  • 各港口壓力變化比較

7.6 系統實作工具

  • 資料分析：R（dplyr, tidyr）
  • 視覺化：ggplot2
  • 互動系統：Shiny

本儀表板將作為一套「可操作的決策支援工具」，而非僅靜態圖表展示。

八、預期價值

8.1 政策價值

本研究可協助政府與港務公司以量化方式辨識港口空櫃壓力，提升跨港調度之預警與協調能力。

8.2 營運價值

本研究可協助辨識空櫃累積風險，降低港口因空櫃滯留所造成之堆場壓力與作業效率損失。

8.3 決策價值

本研究將原先偏經驗導向之空櫃調度判斷，轉化為可視化、可比較、可排序之資料驅動決策流程。

8.4 研究價值

本研究嘗試將空櫃不平衡問題轉化為以公開資料支撐之 proxy-based decision model，作為航港資料分析與決策支援之應用案例。

未來可進一步導入 Prophet 等時間序列方法，作為月別風險預警之延伸模組，但本研究主體仍以跨港調度決策支援為核心。

十、資料庫連結與欄位對照表

本研究使用之資料主要來自 iMarine 航港資料庫、政府資料開放平台與主計總處統計資料。以下說明各資料集之用途、關鍵欄位與轉化邏輯。

10.1 空櫃流動資料（核心資料）

資料集：臺灣進口貨空櫃

來源：https://data.gov.tw/dataset/171741

主要欄位：

  • 年月
  • 臺灣港
  • 20呎標準貨櫃

用途：

  • 建立空櫃流入量（Empty_In）

轉化邏輯：

  • 依港口與年月聚合
  • 作為空櫃累積分析的輸入端

資料集：臺灣出口貨空櫃

來源：https://data.gov.tw/dataset/171743

主要欄位：

  • 年月
  • 臺灣港
  • 20呎標準貨櫃

用途：

  • 建立空櫃流出量（Empty_Out）

轉化邏輯：

  • 與進口空櫃資料合併
  • 計算空櫃淨流量（Empty_Net = In - Out）

10.2 實櫃資料（輔助資料）

資料集：臺灣出口貨實櫃

來源：https://data.gov.tw/dataset/171742

主要欄位：

  • 年月
  • 臺灣港
  • 20呎標準貨櫃

用途：

  • 描述港口出口活動規模
  • 作為背景分析與合理性驗證

資料集：臺灣進口貨實櫃

來源：https://data.gov.tw/dataset/171298

主要欄位：

  • 年月
  • 臺灣港
  • 20呎標準貨櫃

用途：

  • 描述港口進口貨物流動
  • 補充港口整體貨運結構

10.3 港口作業量資料（壓力基準）

資料集：國際商港貨櫃裝卸量（TEU）

來源：iMarine / data.gov.tw（依實際資料頁）

主要欄位：

  • 年月
  • 港口名稱
  • 貨櫃裝卸量（TEU）

用途：

  • 建立港口整體作業規模（Throughput）

轉化邏輯：

  • 作為空櫃壓力正規化基準
  • 同樣空櫃量在不同港口具有不同壓力意義

10.4 官方統計備援資料

資料集：各國際商港貨櫃裝卸量（EXCEL）

來源：https://www.stat.gov.tw/

用途：

  • 與 iMarine 資料交叉驗證
  • 補足缺失或異常資料

10.5 資料整合結構（最終分析表）

所有資料將整合為以下結構：

欄位：

  • port（港口）
  • ym（年月）
  • empty_in（空櫃進港）
  • empty_out（空櫃出港）
  • empty_net（淨流量）
  • full_in（實櫃進口）
  • full_out（實櫃出口）
  • throughput（港口作業量）

10.6 衍生指標（供模型使用）

  1. 空櫃淨流量
    empty_net = empty_in - empty_out
  2. 空櫃壓力指標
    以空櫃量與港口作業量進行正規化
  3. 承接緩衝指標
    buffer = 1 - pressure

用途：

  • 辨識高壓力港（需釋放）
  • 辨識高緩衝港（可承接）

10.7 與研究方法之對應關係

分析階段 使用資料
空櫃流動分析 空櫃進出口資料
港口壓力計算 吞吐量資料
狀態分類 空櫃 + 吞吐量
媒合決策 分類結果 + 區域權重

本資料設計確保所有分析皆可由公開資料完成，避免依賴航商內部訂單或不可取得資料。
